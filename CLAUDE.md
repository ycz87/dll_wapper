## Project Overview

Simulink-to-ADPSS DLL wrapper project. MATLAB/Simulink models (`.slx` files) generate C code via Simulink Coder (ERT target). A Python script (`generate_adpss_wrapper.py`) auto-generates the wrapper that adapts the Simulink interface to ADPSS simulation software's standard C DLL interface.

## Workflow: New Simulink Model

```bash
# 1. Copy or regenerate Simulink code into *_ert_shrlib_rtw/ directory
# 2. Run the generator (auto-detects the directory, or specify it)
python generate_adpss_wrapper.py
python generate_adpss_wrapper.py my_new_model_ert_shrlib_rtw  # explicit

# 3. Build locally (requires MinGW64 GCC on PATH)
cmake -B build -G "MinGW Makefiles"
cmake --build build

# 4. Or push to GitHub — Actions auto-builds and uploads the DLL artifact
```

## Architecture

### Two-Layer Design
1. **Simulink-generated code** (`testadpssmodel260402_ert_shrlib_rtw/`) — auto-generated, do NOT hand-edit
   - `_initialize()`, `_step()`, `_terminate()` + global I/O structs (`_U`, `_Y`)
2. **ADPSS wrapper** (`adpss_wrapper.c`) — auto-generated by `generate_adpss_wrapper.py`
   - Exports: `Inittestadpssmodel260402`, `Steptestadpssmodel260402`, `Terminatetestadpssmodel260402`
   - Maps `Val_In[0]` ↔ `testadpssmodel260402_U.In1`, `Val_Out[0]` ↔ `testadpssmodel260402_Y.Out1`

### ADPSS Interface (Store variant)
```c
int DLLAPI Init$MODEL(void **pModelInfo, int Parameter_N, int Nin, int Nout,
    double *Parameter, double *Val_In, double *Val_out,
    int Store_Size, char *Store_Add, double Tnow, double DT);

int DLLAPI Step$MODEL(void *pModelInfo, int Mark,
    double *Val_In, double *Val_Out, char *Store_Add, double Tnow);

void DLLAPI Terminate$MODEL(void *pModelInfo);
```

### Key Files

- `generate_adpss_wrapper.py` — parses Simulink header, generates wrapper + def + CMakeLists
- `adpss_wrapper.c` — ADPSS ↔ Simulink bridge (auto-generated, do not hand-edit)
- `adpss_wrapper.def` — DLL export definitions (auto-generated)
- `CMakeLists.txt` — build configuration (auto-generated)
- `*_ert_shrlib_rtw/` — Simulink-generated code (read-only)

## Important Notes

- Generated code is single-instance (`MULTI_INSTANCE_CODE=0`); `pModelInfo` is a lightweight wrapper struct, actual state lives in Simulink globals.
- When the Simulink model changes, re-run `python generate_adpss_wrapper.py` — it auto-detects inputs/outputs from the header.
- Build targets Windows x86-64 only.
